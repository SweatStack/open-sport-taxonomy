This is OST inside an app: tag each activity with its canonical sport string, and hierarchical filtering comes for free. A filter matches a sport and every sport beneath it in the dot hierarchy, so cycling selects cycling.road, cycling.gravel, cycling+stationary, and the rest; cycling.road narrows to road rides, and cycling+stationary to the indoor variants.     OpenSportTaxonomy is maintained by SweatStack. We built it because every fitness app ends up rebuilding the same sport plumbing: naming activities, reconciling them across platforms, and keeping a hierarchy straight. It is the vocabulary we wished existed, and we publish it openly because a shared vocabulary only works when it is actually shared. The spec, mappings, and reference implementation are on GitHub.
