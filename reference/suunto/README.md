# Suunto — Activities

> Reference documentation for Suunto's activity classification.

**Source:** `Activities.pdf` (this directory), from Suunto's developer documentation.
Suunto publishes the activity list as a PDF table; there is no machine-readable enum or
swagger for the activity IDs.

## Versioning

The activity list is **not versioned** in the PDF. We pin `platform_version` in
`mappings/suunto.yaml` to the source + fetch date:
`"Activities reference PDF (fetched 2026-06-08)"`. The newest activities (IDs 103–121)
are annotated "since March 2025", so the bundled list is at least that recent. When
refreshing, replace `Activities.pdf`, re-transcribe new rows into `activities.yaml`,
and bump the fetch date.

## Target: the activity ID

A Suunto move carries an integer **activity ID** (0–121). **The integer ID is the
mapping target.** The activity *name* is not unique — IDs 4–9 are all "Sports" — so it
cannot be the key; it is kept only as the row label/comment. The numbering has gaps
(ID 89 is unassigned), like Wahoo's `workout_type_id`.

## The FIT columns are learn-from-only

The PDF's right-hand columns are **Suunto's own** export to Garmin FIT
(`sport` / `sub_sport`). We **learn from** them — they reveal how Suunto conceives each
activity's discipline/indoor/assist character — but **map to OST, not to FIT.** OST is
frequently finer than Suunto's FIT export:

- E-mtb → `cycling.mountain+assisted` (Suunto FIT: `E_BIKING`/E_BIKE_MOUNTAIN — one
  axis; OST keeps discipline + e-assist).
- Roller skiing → `xc_skiing+roller` (Suunto FIT: `CROSS_COUNTRY_SKIING` — roller lost).
- Gravel cycling → `cycling.gravel`; Cyclocross → `cycling.cyclocross`.

And in a few places OST deliberately diverges (e.g. backcountry/telemark skiing →
`null`, since OST models only `xc_skiing`, not alpine touring). See
`plans/020-suunto-mapping.md` for the full rationale and the long `null` tail (Suunto's
taxonomy covers far more than OST models).

## Files

- [`Activities.pdf`](Activities.pdf) — source document (Suunto developer docs).
- `activities.yaml` — hand-curated `{value, name}` for all 121 activity IDs,
  transcribed from the PDF. **Source of truth.** (Added during implementation; see
  `plans/020-suunto-mapping.md`.)
- `targets.yaml` — generated by `scripts/build_reference/suunto.py` from
  `activities.yaml`; the flat list of legal activity-ID targets. Do not edit by hand.
  (Added during implementation.)
