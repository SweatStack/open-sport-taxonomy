# Wahoo Cloud API — Workout Types

> Reference documentation for Wahoo's workout type classification.

**Source:** [Wahoo Cloud API — Data Types](https://cloud-api.wahooligan.com/#data-types)

## Versioning

The Wahoo Cloud API is **not semantically versioned** — there is no SDK version or
spec number to pin. The only published version signal is the **changelog at the bottom
of the docs page** (https://cloud-api.wahooligan.com/, "Changelog" section).

We therefore track the **most recent changelog date** as `platform_version` in
`mappings/wahoo.yaml`, formatted as `"Cloud API (changelog YYYY-MM-DD)"`. When
refreshing this reference, read the latest changelog date from the docs and bump the
`platform_version` to match.

- **Tracked changelog date:** `2025-10-06`
- Recent entries for context: 2025-10-06 (Plans section), 2025-07-22 (deauthorization),
  2025-06-11 (`manual`/`edited`/`name` workout flags), 2025-04-01, 2025-03-04. None
  since 2025-03 changed the workout type enumeration, but the date is what we pin.

## Overview

Wahoo classifies every workout with a numeric `workout_type_id`. Each id also carries
two **derived** attributes that are fixed properties of the type, not independent
workout fields:

- `workout_type_location_id` — INDOOR (0) / OUTDOOR (1) / UNKNOWN (255)
- `workout_type_family_id` — BIKING (0), RUNNING (1), SWIMMING (2), WATER SPORTS (3),
  SNOW SPORTS (4), SKATING (5), GYM (6), WALKING (9), N/A (30), OTHER (31), UNKNOWN (255)

Because location and family are fully determined by `workout_type_id`, **only
`workout_type_id` is used as the mapping target.** Location/family are kept here and in
the per-row mapping comments purely for human review and auditing (e.g. "every INDOOR
type should map to a sport carrying `stationary`, or to `null`/`generic`"). See
`plans/018-wahoo-mapping.md` for the full rationale.

## Key Characteristics

- **Flat integer enum** — no hierarchy in the id itself; the id encodes discipline,
  indoor/outdoor, virtual, and e-assist all in one value.
- **Modifiers as distinct types** — Wahoo expresses OST modifiers as separate ids:
  `BIKING_INDOOR` (12, `+stationary`), `BIKING_INDOOR_VIRTUAL` (68,
  `+stationary+virtual`), `EBIKING` (64, `+assisted`), `RUNNING_RACE` (67, `+race`),
  treadmill ids (`+stationary`), etc.
- **Broader sport coverage than OST** — snow sports beyond XC, water sports, skating,
  golf, yoga, multisport, etc. have no OST equivalent and map to `null`.
- **Sparse numbering** — ids 24, 48, 50–55, 57–60 are unassigned; `255` is UNKNOWN.

## Files

- [`workout_types.yaml`](workout_types.yaml) — hand-curated enum: every
  `workout_type_id` with its `name`, `family`, and `location`. **Source of truth**,
  transcribed from the HTML docs (Wahoo publishes no machine-readable enum).
- [`targets.yaml`](targets.yaml) — generated by
  `scripts/build_reference/wahoo.py` from `workout_types.yaml`; the flat list of legal
  `workout_type_id` targets. Do not edit by hand.
